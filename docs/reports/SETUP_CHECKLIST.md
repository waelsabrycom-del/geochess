## 🚀 خطوات البدء الأولى

### الخطوة 1: التحضير
```bash
# ✅ تأكد من أنك في مجلد المشروع
cd d:\geographical_chess_gameplay_board

# ✅ تثبيت المتطلبات (إذا لم تكن مثبتة)
npm install
```

### الخطوة 2: الفحص الأولي
```bash
# ✅ تشغيل أداة التشخيص
npm run diagnose

# يجب أن تظهر جميع الفحوصات بـ ✓
```

### الخطوة 3: التحسين
```bash
# ✅ تحسين قاعدة البيانات
npm run optimize

# يجب أن ترى: "✓ تم التحسين بنجاح"
```

### الخطوة 4: البدء الأساسي (اختبار)
```bash
# ✅ اختبار بدء عادي
npm start

# يجب أن ترى: "✓ الخادم يعمل على http://0.0.0.0:3000"

# ثم أوقفه: Ctrl+C
```

### الخطوة 5: البدء الكامل (الاستخدام الفعلي)
```bash
# ✅ البدء الكامل (الموصى به)
npm run start:advanced

# اختر: 4️⃣ (الخيار الكامل)

# يجب أن ترى:
# - نافذة الأولى: السيرفر يعمل
# - نافذة الثانية: المراقب يراقب
```

## 📊 التحقق من النجاح

### بعد البدء الكامل، تحقق من:

- [ ] السيرفر يعمل (Terminal الأول)
- [ ] المراقب يعمل (Terminal الثاني)
- [ ] الملفات الجديدة تنشأ:
  - `server-health.log` - سجل الفحوصات
  - `server-stats.json` - إحصائيات السيرفر

### فحص السجلات:
```bash
# في Terminal جديدة
tail -f server-health.log

# يجب أن ترى رسائل مثل:
# ✓ فحص صحة السيرفر: نجح (1/1)
# ✓ فحص صحة السيرفر: نجح (2/2)
```

### فحص الإحصائيات:
```bash
# في Terminal جديدة
type server-stats.json

# يجب أن ترى بيانات JSON تحتوي على:
# "healthChecks": رقم يزيد
# "successfulChecks": نفس الرقم تقريباً
```

## ⚠️ استكشاف الأخطاء

### المشكلة: "Command not found"
```bash
# تأكد من أنك في مجلد المشروع:
cd d:\geographical_chess_gameplay_board

# أعد المحاولة
npm run start:advanced
```

### المشكلة: "Port 3000 is already in use"
```bash
# إما أوقف البرنامج السابق
# أو استخدم port مختلف:
PORT=3001 npm start
```

### المشكلة: "Database locked"
```bash
# شغّل محسّن قاعدة البيانات
npm run optimize

# ثم أعد التشغيل
npm start
```

```bash

# شغّل الخيار الكامل:
npm run start:advanced
# اختر 4️⃣

# انتظر 1-2 دقيقة
# ثم تحقق من الملفات الجديدة
```
