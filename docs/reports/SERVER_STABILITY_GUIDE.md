# 🔧 دليل حل مشاكل استقرار السيرفر

## المشكلة الأساسية
السيرفر يفصل بشكل متكرر (الانقطاع المتكرر)

## الحلول المطبقة

### 1️⃣ تحسينات قاعدة البيانات (database.js)
```
✓ تفعيل Foreign Keys للتحقق من السلامة المرجعية
✓ تفعيل WAL mode لتحسين الأداء والاستقرار
✓ تكوين حجم الكاش (10000 صفحة)
✓ تعيين timeout للقفل (5 ثواني)
```

### 2️⃣ معالج الأخطاء العام (server.js)
```javascript
// منع توقف السيرفر عند حدوث أخطاء غير متوقعة
process.on('uncaughtException', (err) => {
    console.error('❌ خطأ غير معالج:', err);
    // السيرفر يبقى يعمل
});

process.on('unhandledRejection', (reason, promise) => {
    console.error('❌ رفع وعد غير معالج:', reason);
    // السيرفر يبقى يعمل
});
```

### 3️⃣ الإيقاف الآمن (server.js)
```javascript
// إيقاف آمن عند استقبال إشارات النظام
process.on('SIGTERM', () => {
    server.close(() => {
        process.exit(0);
    });
});

process.on('SIGINT', () => {
    server.close(() => {
        process.exit(0);
    });
});
```

## الأدوات الجديدة

### 📊 مراقب السيرفر (server-monitor.js)
يراقب صحة السيرفر كل 30 ثانية ويحتفظ بسجلات التحقق

```bash
# التشغيل
node server-monitor.js

# النتيجة
✓ فحص صحة السيرفر: نجح
✓ يتم تسجيل كل فحص في server-health.log
✓ يتم حفظ الإحصائيات في server-stats.json
```

### 🔧 محسّن قاعدة البيانات (database-optimizer.js)
يقوم بتنظيف وتحسين أداء قاعدة البيانات

```bash
# التشغيل
node database-optimizer.js

# يقوم بـ:
✓ التحقق من سلامة قاعدة البيانات
✓ تحسين قاعدة البيانات (VACUUM)
✓ تنظيف الجلسات المنتهية
✓ حذف المباريات القديمة (أكثر من 90 يوم)
✓ إصلاح الألعاب المعلقة
✓ عرض إحصائيات قاعدة البيانات
```

## خطوات الصيانة الدورية

### يومية:
```bash
# 1. تشغيل السيرفر مع المراقب
npm start &
node server-monitor.js

# 2. مراقبة الأخطاء في server-health.log
```

### أسبوعية:
```bash
# تحسين أداء قاعدة البيانات
node database-optimizer.js
```

### شهرية:
```bash
# تنظيف شامل:
# - قاعدة البيانات
# - السجلات القديمة
# - إعادة تشغيل السيرفر
node database-optimizer.js
```

## تشخيص المشاكل

### إذا استمر السيرفر في الفصل:

1. **تحقق من السجلات**
```bash
# عرض السجلات الحالية
tail -f server-health.log

# البحث عن أخطاء معينة
grep "❌" server-health.log
```

2. **فحص قاعدة البيانات**
```bash
# التحقق من سلامة قاعدة البيانات
node database-optimizer.js

# عرض إحصائيات قاعدة البيانات
sqlite3 database/chess_game.db "SELECT COUNT(*) as games FROM games;"
```

3. **فحص استهلاك الموارد**
```bash
# على Windows
tasklist | find "node"
```

4. **أعد تشغيل السيرفر**
```bash
# أوقف جميع عمليات Node
# ثم أعد التشغيل
npm start
```

## ملاحظات مهمة

⚠️ **تحذيرات:**
- لا تحذف ملف قاعدة البيانات مباشرة
- قم بعمل نسخة احتياطية قبل أي عملية صيانة
- راقب السيرفر لمدة 24 ساعة الأولى بعد التحديث

✅ **أفضل الممارسات:**
- استخدم مراقب السيرفر دائماً
- قم بتحسين قاعدة البيانات أسبوعياً
- احفظ سجلات الأخطاء للمراجعة
- أعد تشغيل السيرفر أسبوعياً على الأقل

## الملفات ذات الصلة

| الملف | الوصف |
|------|-------|
| server.js | الخادم الرئيسي مع معالجة الأخطاء |
| database.js | قاعدة البيانات مع التحسينات |
| server-monitor.js | أداة مراقبة الصحة |
| database-optimizer.js | أداة تحسين الأداء |
| server-health.log | سجل فحوصات الصحة |
| server-stats.json | إحصائيات الخادم |

## الدعم

إذا واصلت المشاكل:
1. تحقق من سجل الأخطاء (server-health.log)
2. قم بتشغيل محسّن قاعدة البيانات
3. أعد تشغيل السيرفر بالكامل
4. راجع console.log output للأخطاء التفصيلية
