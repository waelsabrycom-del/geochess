# 📊 دليل استقرار السيرفر والمراقبة

## 🚀 البدء السريع

### الطريقة الأولى: الواجهة التفاعلية (الموصى بها)
```bash
npm run start:advanced
```
ستظهر قائمة تفاعلية تسمح باختيار الخيار المطلوب.

### الطريقة الثانية: البدء المباشر
```bash
# بدء السيرفر فقط
npm start

# بدء السيرفر مع المراقبة
npm start:monitor

# تحسين قاعدة البيانات
npm run optimize
```

---

## 🔧 الخيارات المتاحة في الواجهة التفاعلية

### 1️⃣ بدء السيرفر بشكل طبيعي
- يشغل السيرفر بدون أي خيارات إضافية
- مناسب للاستخدام اليومي

### 2️⃣ بدء السيرفر مع تحسين قاعدة البيانات أولاً
- يحسّن قاعدة البيانات أولاً
- ثم يشغل السيرفر
- مناسب بعد فترة استخدام طويلة

### 3️⃣ بدء السيرفر + مراقبة الصحة
- يشغل السيرفر والمراقب معاً
- المراقب يفحص صحة السيرفر كل 30 ثانية
- يحفظ السجلات في `server-health.log`

### 4️⃣ بدء السيرفر + تحسين قاعدة البيانات + المراقبة (الكامل)
- يقوم بكل العمليات
- الخيار الأفضل للاستقرار الأمثل
- مناسب للبيئات الإنتاجية

### 5️⃣ تحسين قاعدة البيانات فقط
- يقوم بتنظيف وتحسين الأداء
- لا يشغل السيرفر

### 6️⃣ مراقبة الصحة فقط
- يراقب السيرفر الذي يعمل بالفعل
- مفيد إذا كان السيرفر قيد التشغيل

### 7️⃣ عرض سجل الأخطاء
- يعرض آخر 50 سطر من السجل
- يساعد في تشخيص المشاكل

### 8️⃣ إعادة تشغيل شاملة
- يوقف جميع عمليات Node
- يحسّن قاعدة البيانات
- يعيد تشغيل السيرفر والمراقب

## 📋 الملفات الجديدة

### `server-monitor.js`
أداة مراقبة متقدمة للسيرفر
- **الميزات:**
  - فحص صحة السيرفر كل 30 ثانية
  - حفظ الإحصائيات في `server-stats.json`
  - تسجيل السجلات في `server-health.log`
  - عرض ملخص الإحصائيات كل دقيقة

**الاستخدام:**
```bash
node server-monitor.js
# أو
npm run start:monitor
```

**المخرجات:**
- `server-health.log` - سجل الفحوصات
- `server-stats.json` - إحصائيات الخادم

---

### `database-optimizer.js`
أداة تحسين قاعدة البيانات
- **الميزات:**
  - التحقق من سلامة قاعدة البيانات
  - تحسين الأداء (VACUUM)
  - تنظيف الجلسات المنتهية
  - حذف المباريات القديمة (>90 يوم)
  - إصلاح الألعاب المعلقة
  - عرض إحصائيات شاملة

**الاستخدام:**
```bash
node database-optimizer.js
# أو
npm run optimize
```

---

### `advanced-startup.js`
واجهة تفاعلية لإدارة السيرفر
- **الميزات:**
  - قائمة بصرية سهلة الاستخدام
  - خيارات متعددة للبدء
  - إدارة العمليات تلقائياً
  - إعادة تشغيل شاملة

**الاستخدام:**
```bash
node advanced-startup.js
# أو
npm run start:advanced
```

## 🔍 تشخيص المشاكل

### السيرفر يفصل بشكل متكرر

1. **تحقق من السجلات:**
   ```bash
   # عرض آخر الأخطاء
   tail -n 50 server-health.log
   
   # البحث عن أخطاء معينة
   grep "❌" server-health.log
   ```

2. **حسّن قاعدة البيانات:**
   ```bash
   npm run optimize
   ```

3. **أعد تشغيل شامل:**
   ```bash
   npm run start:advanced
   # اختر 8️⃣ (إعادة تشغيل شاملة)
   ```

### السيرفر يستهلك الكثير من الذاكرة

1. **قم بتحسين قاعدة البيانات:**
   ```bash
   npm run optimize
   ```

2. **افحص الإحصائيات:**
   ```bash
   cat server-stats.json
   ```

### لا يوجد اتصال بالسيرفر

1. **تحقق من التشغيل:**
   ```bash
   # على Windows
   netstat -ano | findstr :3000
   ```

2. **أعد التشغيل:**
   ```bash
   npm run start:advanced
   # اختر 8️⃣
   ```

## ⚙️ التحسينات المطبقة

### في database.js:
✓ تفعيل Foreign Keys
✓ تفعيل WAL mode (أداء أفضل)
✓ تكوين حجم الكاش (10000 صفحة)
✓ تعيين timeout للقفل (5 ثواني)

### في server.js:
✓ معالج أخطاء شامل
✓ معالج uncaughtException
✓ معالج unhandledRejection
✓ إيقاف آمن (SIGTERM/SIGINT)
