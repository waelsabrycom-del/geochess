# ميزة الدردشة بين المدير والقادة

## نظرة عامة
تم إضافة ميزة جديدة تسمح للمدير (منشئ البطولة) بالتواصل مع كل قائد انضم للبطولة بشكل فردي من خلال نافذة دردشة.

## المتطلبات الجديدة
- **المتطلب:** "اكونت المدير حقل القادة المشاركون ضع ايكون الشات امام كل قائد انضم مع امكانية شات بين المدير والقادة منفردين"

## المكونات المضافة

### 1. واجهة المستخدم (UI)
**الملف:** `تفاصيل البطولة والمتابعة.html`

#### أ) نافذة الدردشة (Chat Modal)
- **الموقع:** أسفل العنوان وفوق live-stream-modal
- **المعرفات:**
  - `chat-modal` - الحاوية الرئيسية
  - `chat-participant-name` - اسم المشارك الحالي في العنوان
  - `chat-messages` - منطقة عرض الرسائل
  - `chat-input` - حقل إدخال الرسالة
  
- **الخصائص:**
  - مخفية افتراضياً (hidden class)
  - عند الفتح، تُعرض فوق جميع العناصر الأخرى (z-[150])
  - خلفية داكنة شفافة مع تأثير blur

#### ب) أيقونة الدردشة في بطاقات المشاركين
- **الموقع:** أعلى يسار كل بطاقة مشارك (للمدير فقط)
- **الشروط:**
  - تظهر فقط عندما يكون المستخدم الحالي هو منشئ البطولة
  - تظهر فقط للمشاركين المقبولين (status = 'accepted')
  - تختفي عند الهوفر أولاً ثم تظهر عند الهوفر على البطاقة
  
- **الإجراء:** عند الضغط تستدعي `openChatModal(participantId, participantName)`

### 2. المتغيرات العامة
```javascript
let currentTournamentCreatorId = null;  // معرّف منشئ البطولة
let currentChatParticipantId = null;    // معرّف المشارك الحالي للدردشة
let currentChatParticipantName = null;  // اسم المشارك الحالي للدردشة
```

### 3. الدوال الجديدة

#### أ) `openChatModal(participantId, participantName)`
**الوصف:** فتح نافذة الدردشة مع مشارك محدد

**الخطوات:**
1. تعيين `currentChatParticipantId` و `currentChatParticipantName`
2. تحديث عنوان البطاقة برسالة "دردشة مع [اسم المشارك]"
3. إزالة الـ hidden class من modal
4. تحميل الرسائل السابقة (إن وجدت)

**مثال الاستخدام:**
```javascript
openChatModal(123, 'أحمد علي')
```

#### ب) `closeChatModal()`
**الوصف:** إغلاق نافذة الدردشة

**الخطوات:**
1. إضافة الـ hidden class لإخفاء modal
2. مسح `currentChatParticipantId` و `currentChatParticipantName`
3. مسح محتوى حقل الإدخال

#### ج) `sendChatMessage()`
**الوصف:** إرسال رسالة جديدة

**الخطوات:**
1. الحصول على نص الرسالة من `chat-input`
2. إنشاء كائن رسالة بـ:
   - `senderId` - معرّف المُرسل من localStorage
   - `text` - نص الرسالة
   - `timestamp` - الوقت الحالي
   - `recipientId` - معرّف المستقبل
3. حفظ الرسالة في localStorage
4. تحديث عرض الرسائل
5. مسح حقل الإدخال

#### د) `getChatKey(participantId)`
**الوصف:** الحصول على مفتاح التخزين الفريد لدردشة معينة

**الصيغة:** `chat_[userId]_[participantId]_[tournamentId]` (مرتبة أبجدياً)

**الفائدة:** تضمن أن كل دردشة بين مستخدمين في بطولة لها مفتاح فريد

#### هـ) `loadChatMessages()`
**الوصف:** تحميل وعرض الرسائل السابقة

**الخطوات:**
1. الحصول على المفتاح الفريد للدردشة
2. استرجاع الرسائل من localStorage
3. عرض الرسائل مع:
   - محاذاة يمين للرسائل المرسلة من المستخدم الحالي
   - محاذاة يسار للرسائل المرسلة من المشارك
   - ألوان مختلفة (أزرق للمرسل، رمادي للمستقبل)
   - الوقت بصيغة عربية (مثل 3:45 ص)

#### و) `escapeHtml(text)`
**الوصف:** تنظيف النص لحماية من هجمات XSS

**الأحرف المُعالجة:**
- `&` → `&amp;`
- `<` → `&lt;`
- `>` → `&gt;`
- `"` → `&quot;`
- `'` → `&#039;`

## طريقة التخزين

### الموقع: localStorage
كل دردشة تُخزّن كمصفوفة JSON تحت المفتاح: `chat_[userId]_[participantId]_[tournamentId]`

### هيكل الرسالة:
```javascript
{
    senderId: "123",                           // معرّف المُرسل
    senderName: "أحمد علي",                     // اسم المُرسل
    recipientId: "456",                        // معرّف المستقبل
    text: "مرحبا! كيف حالك؟",                   // نص الرسالة
    timestamp: "2024-01-15T10:30:45.123Z"     // الوقت بصيغة ISO
}
```

### مثال التخزين في localStorage:
```javascript
localStorage.getItem('chat_123_456_789')
// Returns: [{"senderId":"123",...}, {"senderId":"456",...}]
```

## تدفق الاستخدام

### خطوات المستخدم:
1. **فتح صفحة تفاصيل البطولة**
   - يتم تحميل تفاصيل البطولة بما فيها معرّف منشئ البطولة

2. **رؤية القائمة الكاملة للمشاركين**
   - إذا كان المستخدم هو منشئ البطولة، سيرى أيقونة دردشة زرقاء على كل مشارك

3. **الضغط على أيقونة الدردشة**
   - تظهر نافذة دردشة بعنوان "دردشة مع [اسم القائد]"
   - يتم تحميل الرسائل السابقة (إن وجدت)

4. **كتابة الرسالة والإرسال**
   - يكتب الرسالة في حقل الإدخال
   - يضغط Enter أو يضغط زر الإرسال
   - تظهر الرسالة على اليمين بلون أزرق

5. **رؤية الرسائل السابقة**
   - عند فتح الدردشة مرة أخرى مع نفس القائد، تظهر جميع الرسائل السابقة

6. **إغلاق الدردشة**
   - يضغط زر X أو يضغط خارج النافذة
   - تختفي النافذة والرسائل تبقى محفوظة

### المتصفحات المدعومة:
- ✅ Chrome/Edge (v90+)
- ✅ Firefox (v88+)
- ✅ Safari (v14+)

### الميزات المستخدمة:
- `localStorage` - معايير HTML5
- `JSON.parse/stringify` - معايير ES5
- `classList.add/remove` - معايير DOM جديدة
- `toLocaleTimeString` - معايير Intl API

### حدود معروفة:
- الرسائل محفوظة محلياً فقط (تُفقد عند مسح بيانات المتصفح)
- لا تتم مزامنة الرسائل بين أجهزة مختلفة
- الحد الأقصى لحجم التخزين يعتمد على حد localStorage (عادة 5-10MB)
